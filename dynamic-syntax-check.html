dynamic syntax check

發表於 2022-04-01 分類於 Keil-MDK

編譯時既沒有錯誤也沒有警告，但是 Keil MDK 的編輯界面卻出現紅色叉號！這一切的背後，是編譯器的扭曲，還是語法器的淪喪？


這兩天創建新項目的時候，在代碼中包含了 CMSIS_5 中 DSP 模塊下的 arm_math.h 文件：

然後發現在 Keil MDK 的編輯界面中該行代碼的左側出現了一個紅色的叉號，提示類型衝突：

可是程序編譯時既沒有錯誤也沒有警告！怪哉怪哉！

進到 none.h 和 cmsis_armcc.h 中看了一下，這個警告貌似是因爲 cmsis_armcc.h 與 none.h 中 __USAT 的定義不一致造成的：

cmsis_armcc.h :

none.h :

不過這個說法有两个疑点：

  1. 在 none.h 中，函數 __USAT 的外面由 #if defined ( _MSC_VER ) || defined(__GNUC_PYTHON__) 宏包着，而程序以及環境中並沒有 _MSC_VER 或 __GNUC_PYTHON__ 的定義，也就是說這段代碼本該是透明的。但是其實際表現卻與理論相悖，因爲我删除任何一個 __USAT 定義紅色叉號均會消失。
  2. 與 __USAT 類似的還有一個 __SSAT 函數，爲什麼 __SSAT 就沒有出現這個問題？

經過一番查閱，我找到了一個近似的問題：

SYMPTOM

On the line of an include statement for a system header file, the Dynamic Syntax Checker has marked a red X. If I hover over the X, it says: “error in include chain (cmsis_armcc.h): expected identifier or ‘(‘“. Fortunately, when I build, the project compiles without error.

CAUSE

This was a false positive reported by the Dynamic Syntax Checker. It is safe to ignore.

RESOLUTION

Install a later version of Keil MDK. Or, the article listed under “SEE ALSO” describes a workaround, to remove the warning.

If the include file exists at the correct location but the issue persists, edit the file UVCC.ini found in the Keil Installation folder “C:\Keil_v5\UV4” to ignore specific lines of code or whole source files. The top of UVCC.ini includes instructions on how to edit this file. This works well for unsupported source code in a C/C++ file, for example, embedded assembly code.

其中指出，該問題是由動態語法檢查器（Dynamic Syntax Checker）報告的一個假陽性錯誤造成的，可以無視且不會存在安全隱患。當然你也可以選擇解決它，下面給出了具體的解決方案：

方法一：安裝最新版本的 Keil MDK 軟件。

方法二：在 Keil MDK 安裝目錄下的 /UV4/UVCC.INI 文件中配置忽略對某行代碼或文件的語法檢查。
